核心问题：企业 AI 的"失忆"困境

7周失忆事故

企业部署 AI Agent 后，客户发现问题并持续反馈了 7 周，Agent 始终无法记住这些反馈——每次对话都从零开始。这是"记忆缺失"不是"模型不够好"导致的系统性失败。

诊断：不是模型问题，是基础设施缺位。记忆是 AI 从 Demo 到生产的关键缺失层。

核心定义：记忆 ≠ 存储

记忆 = 理解 + 存储 + 组织 + 推理 + 遗忘 + 演进

记忆是活的：不只是写入/读取，还包括压缩、归档、关联、衰减、迭代更新。这与 RAG 的静态检索根本不同。

六种记忆类型

MemoryLake 对记忆按内容语义分类为六种类型：

类型内容示例
Background用户背景信息职业、偏好、历史
Facts可核实事实名称、规则、参数
Events时间序列事件决策记录、会议纪要
Conversations对话历史摘要聊天压缩、关键问答
Actions执行行为记录操作日志、工具调用
Reflections推理与洞察从经验归纳出的规律

五级记忆层次结构

按时间尺度和持久性分五层：

层级名称时间尺度特点
L1工作记忆当前请求（秒级）活跃上下文
L2短期记忆当前会话（分钟）会话连续性
L3中期记忆数天~数周项目/任务记忆
L4长期记忆数月~数年个性化知识
L5场景意识记忆永久角色定义、核心规则

三层记忆粒度管理

让每个 Agent 都有自己的"脑"：

层级内容共享策略
Instance 级全局共享的组织记忆所有 Agent 可读
Agent 级每个 Agent 的私有记忆角色定义、专属知识
Session 级每次会话的临时记忆上下文、短期事实

任意维度可配置共享/隔离，通过策略矩阵灵活控制。

游戏 AI 示例：

  • Instance：世界观设定（所有 NPC 共享）
  • Agent：NPC 角色记忆（每个 NPC 独立）
  • Session：玩家本次对话上下文（会话结束后压缩入 Agent 记忆）

MemoryLake 三层架构

Skills Center（技能层）
  ↓ 调用
LakeBuilder（记忆构建层）— 摄入、理解、组织、索引
  ↓ 存储
QueryAgent（记忆查询层）— 多策略检索、多跳推理

三方索引融合（支撑超大规模检索）：

  • 向量索引 + BM25 关键词 + 知识图谱
  • 目标：10万亿条记录下仍 <200ms 响应

三个生产硬门槛

  1. 多模态精准理解：文档、图片、音视频、表格的统一语义理解
  2. 超大规模低成本检索：10T 记录下向量检索会超时，需三方索引融合 + 分布式计算
  3. 可解释可干预：Memory Time Travel（完整记忆溯源链）+ 企业信任三角（准确性/隐私/合规）

生产数据：

  • 1亿+ 文档，10万亿+ 记录
  • 向量检索在 10T 量级超时 → 三方融合方案实现 <200ms + 30x 成本下降

核心论点（结论页）

  1. 记忆是 AI 从 Demo 到生产的关键缺失层 — 不是模型不够好，是基础设施缺位
  2. 记忆 ≠ 存储 — 记忆是活的：理解、组织、压缩、遗忘、演进
  3. 生产级记忆工程三个硬门槛：多模态精准理解、超大规模低成本检索、可解释可干预
  4. MemoryLake：全球唯一兼具"记忆 + 平台 + 大规模生产验证"的解决方案（自评）
