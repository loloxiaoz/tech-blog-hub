# 有了大语言模型后，知识图谱该何去何从？

**来源**：知乎（IntrospectorMachines，758赞）  
**原文**：https://www.zhihu.com/question/588865784/answer/2937950896

## 核心判断

KG 没什么用了，是科研界的自嗨。

LLM 出现后，KG 社区无法提出真正有竞争力的论据，只能靠 14 种"讲故事的方法"自我安慰。

## 14 种讲故事的方法（及作者的隐含评价）

| 方法 | 论点 | 实际成色 |
|------|------|---------|
| 指鹿为马法 | "LLM 就是 KG" | 偷换概念 |
| 只因你太美法 | KG 做 PPT 好看 | 无关技术价值 |
| 可解释性强法 | 小图可解释 → 大图也可解释 | 以偏概全 |
| 可靠性强法 | 确定性逻辑 | 前提"用户输入可靠"存疑 |
| 节能环保法 | KG 省电 | LLM 可量化蒸馏，差距在缩小 |
| 节省硬盘法 | 结构熵编码后很小 | 工程优化，非核心竞争力 |
| 启发人类法 | 人工标注效率高 | CoT 已能启发人类 |
| 校正先验法 | KG 推理出 groundtruth 校正 LLM | 循环论证 |
| Prompt 工程法 | KG 结果作 prompt 喂 LLM | 有一定可行性，但 KG 沦为配角 |
| 可视化法 | 微调让 LLM 注意力可视化为 KG | 又绕回"LLM 就是 KG" |
| 便于编辑法 | 可视化界面编辑，非技术人可用 | 唯一比较实在的差异化 |
| 百万漕工法 | 保护就业，少用 LLM | 与技术无关 |
| 哲学意义法 | Ontology 有哲学价值 | 学术圈自我欣赏 |
| 因噎废食法 | LLM 有安全漏洞，KG 只能增删查改更安全 | 以缺陷为卖点 |
