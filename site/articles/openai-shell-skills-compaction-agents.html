Shell + Skills + Compaction: Tips for long-running agents that do real work

2026-02-11约 5 分钟AgentSkillsCompaction
Summary
OpenAI 发布三项 Agentic 原语：Skills（可复用的版本化指令包）、升级版 Shell 工具（托管容器 + 受控网络访问）、以及服务端 Compaction（自动压缩长对话历史）。三者结合可大幅提升长任务 Agent 的可靠性和可预测性。
原文链接
https://developers.openai.com/blog/shell-skills-compaction-tips-for-long-running-agents

我们正在从单轮助手转向能处理真实知识工作的长任务 Agent：读取大型数据集、更新文件、编写应用。基于开发者反馈和构建 Codex 及内部 Agent 的经验，OpenAI 发布了一套新的 Agentic 原语。

A brief mental model

Skills: 模型可按需加载的"程序"

Skill 是包含 SKILL.md 清单的文件包，内含前置元数据和指令。可以理解为一个版本化的操作手册，模型需要做实际工作时可以随时查阅。

Shell tool: Agent 的"执行环境"

Shell 工具让模型在真实终端环境中工作，支持：

  • OpenAI 托管的容器
  • 你自己执行的本地 Shell 运行时

Compaction: 保持长任务运行

服务端 Compaction 通过自动管理上下文窗口和压缩对话历史来保持长任务的连续性运行。

Why they're better together

  • Skills 将稳定的程序和示例移入可复用包，消除"提示词意大利面"
  • Shell 提供完整执行环境，可安装代码、运行脚本、写入输出
  • Compaction 在长任务中保持连续性，同一工作流可持续执行，无需手动上下文手术

Tips and tricks

1. 像写路由逻辑一样写 Skill 描述（不是营销文案）

Skill 的描述实际上是模型的决策边界。它应该回答：什么时候该用？什么时候不该用？输出和成功标准是什么？

2. 添加反例和边界情况以减少误触发

Glean 发现基于 Skill 的路由最初在针对性评测中触发率下降约 20%，在描述中添加反例和边界情况后才得以恢复。

3. 将模板和示例放入 Skill（不用时几乎零成本）

Skill 内的模板和可行示例有两个优势：在需要时精准可用；不会为无关查询增加 token 膨胀。

4. 从设计之初就考虑长任务运行

  • 跨步骤复用同一容器
  • 传递 previous_response_id 让模型续接工作
  • 将 Compaction 作为长任务的默认原语，而非应急备选

5. 需要确定性时，明确告诉模型使用某个 Skill

直接说："使用 <skill name> skill。"这将模糊路由变为明确合约。

6. 将 Skills + 网络访问视为高风险组合（为隔离而设计）

强烈建议的默认安全姿态：Skills 允许 / Shell 允许 / 网络 仅通过最小白名单按请求启用。

7. 将 /mnt/data 作为制品的交接边界

将 /mnt/data 视为写入输出的标准位置，用于检索、审查或传回。

8. 理解白名单的双层系统

  • 组织级白名单（由管理员配置）：设置允许的最大目标范围
  • 请求级 network_policy：必须是组织白名单的子集

9. 使用 domain_secrets 进行身份验证调用

如果允许的域需要 Auth headers，使用 domain_secrets，这样模型永远不会看到原始凭证。

10. 在云端和本地使用相同的 API

Skills 在托管 Shell 和本地 Shell 模式下都可使用。跨两种模式保持 Skills 一致。

Three build patterns

Pattern A: Install → fetch → write artifact

从托管 Shell 获益的最简单方式：安装依赖、获取外部数据、生成具体交付物。

Pattern B: Skills + shell for repeatable workflows

将工作流编码在 Skill 中，挂载到 Shell 环境，让 Agent 遵循 Skill 确定性地生成制品。

Pattern C (advanced): Skills as enterprise workflow carriers

Salesforce 导向的 Skill 在 Glean 的评测中将准确率从 73% 提升至 85%，同时将首 token 时间减少了 18.1%。

Build once, run anywhere

  • 用 Skills 编码"如何做"（程序、模板、护栏）
  • 用 Shell 执行"去做"（安装、运行、写入制品）
  • 用 Compaction 保持长任务连贯性
  • 快速迭代时在本地开始；需要可重复的隔离执行时迁移到托管容器
  • 用组织级和请求级白名单锁定网络访问
