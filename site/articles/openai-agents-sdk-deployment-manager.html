Agents SDK Deployment Manager

2026-05-05约 4 分钟Agents SDK部署Docker
Summary
Agents SDK Deployment Manager 是一款本地控制平面应用，用于运行和观察 Agents SDK 演示项目。支持 Docker 容器化部署、本地链路追踪捕获、应用日志和 Span 事件可视化，无需依赖 OpenAI 平台仪表板即可获得完整的可观测性。
原文链接
https://cookbook.openai.com/examples/agents_sdk/deployment_manager/README

概述

这是一款本地控制平面应用，用于运行和观察 Agents SDK 的演示项目。

先决条件

  • uv
  • npm
  • Docker（使用默认 local-docker 目标时需要）

运行方式

make run

启动后访问地址：http://127.0.0.1:8732

说明：Vite 会将 frontend/ 目录下的 React UI 构建到 dist/ 目录，Flask 后端会同时提供 dist/ 静态资源和 /api/* 接口服务。

功能范围

  • 导入本地 Agents SDK 项目
  • 检查入口文件、依赖项、环境变量和沙箱使用情况
  • 创建本地部署记录
  • 默认情况下，将应用/编排器作为带标签的 Docker 容器启动和停止
  • 生成或复用应用级 Dockerfile 用于容器化部署
  • 展示部署日志、链路追踪、应用事件和 Docker 容器活动
  • 为沙箱容器添加 agents-sdk.* 元数据标签，让管理器可以将 Docker 容器映射回对应部署和运行实例

Docker 部署的默认目标是构建应用 Dockerfile，并通过以下命令运行编排器：

docker run -p 127.0.0.1:<app-port>:<app-port> -v /var/run/docker.sock:/var/run/docker.sock ...

挂载 Docker 套接字允许 SandboxAgent 通过 Docker 创建沙箱容器。

管理器会在应用进程上设置以下环境变量：

  • AGENTS_SDK_MANAGER
  • AGENTS_SDK_PROJECT
  • AGENTS_SDK_DEPLOYMENT_DATA_DIR
  • AGENTS_SDK_MANAGER_TRACE_ENDPOINT

可部署应用约定

通过本管理器部署的应用需要满足以下要求：

  • pyproject.toml 文件中包含 openai-agents 依赖
  • 以 main.py 作为应用入口文件
  • 支持 PORT 环境变量用于本地启动
  • 当 PORT 变量被设置时，执行 uv run python main.py 可以启动 HTTP 服务
  • 提供 /health 健康检查端点

链路追踪

当 Agents SDK 仍然通过默认处理器向 OpenAI 平台流式传输链路数据时，本管理器会通过自身的 HTTP 摄取端点本地捕获链路数据。

管理器启动应用时，会设置以下环境变量：

AGENTS_SDK_MANAGER_TRACE_ENDPOINT=http://<manager-host>:8732/api/traces/ingest
AGENTS_SDK_DEPLOYMENT_ID=<deployment-id>
AGENTS_SDK_PROJECT_ID=<project-id>

运行时包使用 sitecustomize.py 在应用进程内安装轻量级的 Agents SDK 链路处理器，通过 add_trace_processor() 实现。这会在不替换 SDK 默认 OpenAI 平台导出器的情况下，添加第二个本地目标。本地处理器会将导出的链路记录回传给管理器，包含以下事件类型：

  • trace_start（链路开始）
  • trace_end（链路结束）
  • span_start（跨度开始）
  • span_end（跨度结束）

管理器会将链路数据存储在 state/traces.sqlite3 中。UI 使用这些重构的记录来实现：链路追踪视图、嵌套跨度列表、事件详情面板、通过 ?trace_id=... 访问的链路深链、JSON 导出。

这意味着本地链路追踪无需将链路文件绑定挂载到应用容器中，也无需从 OpenAI 平台仪表板 API 获取跨度数据，即可正常工作。

Make 命令

# 启动管理器
make start
# 检查管理器健康状态
make health
# 部署到 Docker 环境
make deploy PROJECT_PATH=/path/to/agents-sdk-app APP_PORT=8421 SANDBOX_BACKEND=docker
# 部署到本地进程环境
make deploy PROJECT_PATH=/path/to/agents-sdk-app APP_PORT=8421 TARGET=local-process
# 停止应用
make stop
