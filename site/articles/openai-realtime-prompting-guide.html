Realtime Prompting Guide

2026-02-25约 15 分钟RealtimePromptingVoice

原文链接：https://cookbook.openai.com/examples/Realtime_prompting_guide

概述

今天，我们发布 gpt-realtime——API 中迄今最强大的语音转语音模型，并宣布 Realtime API 正式可用。

新的 gpt-realtime 模型提供更强的指令跟随能力、更可靠的工具调用、明显更好的语音质量，以及整体更流畅的感觉。这些改进使从链式方法转向真正的实时体验变得实际可行，降低延迟并产生听起来更自然、更富表现力的响应。

通用提示技巧

  • 不断迭代：细微的措辞变化可以决定行为的成败。示例：对于不清晰的音频指令，将"inaudible"换为"unintelligible"改善了嘈杂输入处理。
  • 偏好项目符号而非段落：清晰、简短的项目符号优于长段落。
  • 用示例引导：模型会密切遵循示例短语。
  • 精确描述：歧义或相互冲突的指令会降低性能。
  • 控制语言：如果出现不必要的语言切换，将输出固定到目标语言。
  • 减少重复：添加多样性规则以减少机械化措辞。
  • 用大写文本强调：将关键规则大写，使其突出并更容易让模型遵循。

提示词结构

组织提示词使模型更容易理解上下文并在多轮对话中保持一致。建议的提示词骨架：

# Role & Objective        — 你是谁，"成功"意味着什么
# Personality & Tone      — 保持的声音和风格
# Context                 — 检索到的上下文、相关信息
# Reference Pronunciations — 难词的音标指南
# Tools                   — 名称、使用规则和 preamble
# Instructions / Rules    — 做什么、不做什么和方法
# Conversation Flow       — 状态、目标和转换
# Safety & Escalation     — 回退和切换逻辑

角色与目标

此部分定义 Agent 是谁以及"完成"意味着什么。

# Role & Objective
You are a high-energy game-show host guiding the caller to guess a secret number from 1 to 100 to win 1,000,000$.

新模型（gpt-realtime）成功地承担了高能量游戏节目主持人的角色，带来自然、吸引人的表达。

个性与语气

新模型非常擅长遵循指令来模仿特定的个性或语气。

基本示例

# Personality & Tone
## Personality
- Friendly, calm and approachable expert customer service assistant.

## Tone
- Warm, concise, confident, never fawning.

## Length
2–3 sentences per turn.

速度指令

在 Realtime API 中，speed 参数改变播放速率，而不是模型如何组织语音。要真正听起来更快，添加可以引导节奏的指令：

## Pacing
- Deliver your audio response fast, but do not sound rushed.
- Do not modify the content of your response, only increase speaking speed for the same response.

语言约束

## Language
- The conversation will be only in English.
- Do not respond in any other language even if the user asks.
- If the user speaks another language, politely explain that support is limited to English.

减少重复

## Variety
- Do not repeat the same sentence twice.
- Vary your responses so it doesn't sound robotic.

发音参考

确保模型在口语交互中正确发音重要单词、数字、名称和术语：

# Reference Pronunciations
When voicing these words, use the respective pronunciations:
- Pronounce "SQL" as "sequel."
- Pronounce "PostgreSQL" as "post-gress."
- Pronounce "Kyiv" as "KEE-iv."

字母数字发音

对于电话号码、信用卡、订单ID等，明确的逐字符确认可防止误听：

# Instructions/Rules
- When reading numbers or codes, speak each character separately, separated by hyphens (e.g., 4-1-5).
- Repeat EXACTLY the provided number, do not forget any.

工具配置

使用此部分告诉模型如何使用你的函数和工具。说明何时调用工具、何时不调用、要收集哪些参数、调用运行时说什么，以及如何处理错误。

工具调用 Preamble

某些用例可能受益于 Realtime 模型在调用工具的同时提供音频响应，以掩盖延迟：

# Tools
- Before any tool call, say one short line like "I'm checking that now." Then call the tool immediately.

工具输出格式化

某些工具输出（尤其是必须逐字重复的长字符串）可能超出模型的分布范围。实用的解决方法是使工具输出看起来像普通工具结果，并使逐字要求机器可理解：

{
  "response_text": "I just sent you an email with the verification link. Please open it and click \"Confirm\".",
  "require_repeat_verbatim": true
}

对话流程

此部分介绍如何将对话结构化为清晰的、以目标为导向的阶段，以便模型确切知道在每一步做什么。定义每个阶段的目的、通过它的指令和具体的"退出条件"来过渡到下一个阶段。

模式1：状态机对话流

将对话定义为编码状态和转换的 JSON 结构。这使得推理覆盖率变得容易，识别边缘案例，并随时间跟踪变化：

[
  {
    "id": "1_greeting",
    "description": "Begin each conversation with a warm, friendly greeting.",
    "instructions": [
        "Use the company name 'Snowy Peak Boards' and provide a warm welcome."
    ],
    "examples": [
      "Hello, this is Snowy Peak Boards. Thanks for reaching out! How can I help you today?"
    ],
    "transitions": [{
      "next_step": "2_get_first_name",
      "condition": "Once greeting is complete."
    }]
  }
]

模式2：动态对话流

在这种模式中，对话通过根据当前状态更新系统提示和工具列表来实时适应。当状态的退出条件满足时，使用 session.update 转换，用下一阶段所需的提示和工具替换。这减少了模型的认知负荷。

安全与升级

# Safety & Escalation
When to escalate (no extra troubleshooting):
- Safety risk (self-harm, threats, harassment)
- User explicitly asks for a human
- Severe dissatisfaction (e.g., "extremely frustrated," repeated complaints)
- 2 failed tool attempts on the same task OR 3 consecutive no-match/no-input events
- Out-of-scope or restricted requests

What to say at the same time of calling the escalate_to_human tool:
- "Thanks for your patience—I'm connecting you with a specialist now."

新模型（gpt-realtime）可靠地遵循所有升级规则，正确识别何时切换到人工 Agent，并在进行工具调用前发出必要的 preamble。
