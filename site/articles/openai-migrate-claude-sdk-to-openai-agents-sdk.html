从 Claude Agent SDK 迁移到 OpenAI Agents SDK

为什么迁移

全新的 OpenAI Agents SDK 为团队提供了原生模型的 harness，用于构建可跨工具、文件、记忆、审批和沙箱计算协调的 Agent。它是开源的，团队可以检查和自定义运行时行为。

最重要的架构差异：

  • Claude Agent SDK 模式：harness 在沙箱内运行。沙箱包含 Agent 循环、MCP/工具、文件系统访问和执行环境。
  • 新 OpenAI Agents SDK 模式：harness 与计算分离。可信应用运行时拥有 Agent 循环、工具、审批、密钥、护栏、追踪和业务系统访问权限。

到本指南结束时，你应该拥有一个迁移后的应用，在保留原始 Claude 应用行为的同时，使执行、安全、状态和所有权显式化。

架构

沙箱作为工具

将可信控制保留在宿主应用中，仅将沙箱用于以工作区为中心的执行。这更安全、更容易操作，因为密钥、审批决策和业务系统访问都留在执行环境之外。

技能和说明文件

迁移 Claude Agent SDK 项目时，请按职责拆分说明和技能文件。

ClaudeOpenAI迁移规则
CLAUDE.md 产品行为Agent.instructions在此处保留 Agent 角色、任务策略、工具使用规则和响应格式。
CLAUDE.md 仓库或工作区指导AGENTS.md在此处保留仓库范围的约定、测试命令、输出路径和非默认工作区规则。
.claude/skills/*/SKILL.md.agents/skills/*/SKILL.md将可重用工作流转换为 OpenAI 技能。
.claude/agents/*.md显式 Agent(...) 定义为每个 Claude 子 Agent 创建命名 Agent(...) 和清晰的说明。

子 Agent 和所有权

  • 当主导 Agent 应保持控制、调用专家、组合其输出并生成最终答案时，使用 agent.as_tool(...)。
  • 当专家应接管对话并拥有最终响应时，使用 handoffs=[...]。

你需要迁移什么

大多数迁移归结为十个决策：

Claude 概念迁移
沙箱边界决定在隔离计算还是可信运行时中运行什么
说明和技能将 CLAUDE.md / Agent 说明映射到 Agent.instructions；将可重用工作流转换为 OpenAI 技能。
自定义工具将业务/领域操作转换为类型化的 @function_tool 函数
内置工具选择正确的执行面：托管、本地/运行时或基于沙箱
护栏在输入、输出和工具边界添加护栏
Agent 循环使用 Runner.run(...)
多 Agent 所有权使用 agent.as_tool(...) 保持编排器控制，或通过 handoffs=[...] 将控制权传递给专家
对话延续为对话上下文、暂停运行恢复和沙箱会话恢复选择单独的策略
审批将副作用置于人工审批之后，并从中断恢复
生命周期回调将日志记录/追踪移动到 RunHooks 或 AgentHooks。将阻止、审批或执行重塑逻辑移动到护栏、审批或过滤器

分步迁移

步骤 1：定义第一个 OpenAI Agent

如果一个 Claude Agent 端到端处理单个任务，通常映射到一个 OpenAI Agent。

步骤 2：移植自定义工具

按以下顺序移植自定义工具：

  1. 保持业务操作不变。
  2. 收紧函数签名。
  3. 编写解释何时使用工具的函数文档。
  4. 返回 Agent 可以检查的结构化数据。

步骤 3：按执行边界映射内置工具

  • 对业务或领域操作使用 @function_tool。
  • 对 OpenAI 管理的检索或执行使用托管工具。
  • 对偶尔的本地/运行时操作使用 ShellTool 或 ApplyPatchTool。
  • 当任务需要隔离工作区时，使用带有沙箱功能的 SandboxAgent。

步骤 4：将验证移动到正确的护栏边界

需求OpenAI 边界
传入请求必须包含路线、日期和预算输入护栏。
最终答案必须包含推荐或升级输出护栏。
工具调用必须遵守参数约束工具护栏。
操作更改生产状态或花费资金审批和运行状态恢复。

步骤 8：推荐测试顺序

  1. 使用迁移后的自定义工具运行基线 Agent。
  2. 添加输入和输出护栏。
  3. 仅在需要的地方添加工具级检查。
  4. 添加专家 Agent。
  5. 使用 agent.as_tool(...) 或 handoffs=[...] 检查所有权。
  6. 添加对话延续。
  7. 添加副作用审批。

领域通过条件

结论

成功的迁移保留业务任务并更新执行模型：

  • 在隔离重要的地方添加沙箱化。 OpenAI 的 Agents SDK 将可信应用与发生有风险或有状态工作的工作区分开。
  • 将业务操作作为显式工具迁移。 搜索航班或创建票据等操作应成为具有类型化输入和可预测输出的清晰业务工具。
  • 保持工具访问和审批分离。 首先决定 Agent 可以看到哪些工具。然后为需要人工审查的操作添加审批检查。
  • 适当使用护栏和审批。 使用护栏检查请求和输出。使用审批暂停有风险的副作用。
