Testing Agent Skills Systematically with Evals

2026-01-22约 5 分钟EvalsCodexSkills
Summary
系统性评测 Agent Skills 的实用指南：从定义成功标准开始，通过构建 Skill、手动触发验证、小型测试集捕获回归、轻量确定性评分器、到 Codex 辅助的基于 Rubric 评分，形成可持续迭代的评测体系。
原文链接
https://developers.openai.com/blog/testing-agent-skills-systematically-with-evals

当你在迭代一个 Agent（如 Codex）的 Skill 时，很难判断你究竟是在改进它，还是仅仅在改变其行为。Evals（评估）会检查模型的输出以及为产生该输出所执行的步骤，是否与你的预期一致。

具体而言，一次 Eval 就是：一个提示词 → 一次捕获的运行（trace + 制品）→ 一组小型检查 → 一个可随时间比较的评分。

1. 在编写 Skill 之前定义成功标准

将检查项分为四类：

  • 结果目标：任务是否完成？
  • 过程目标：Codex 是否调用了 Skill 并遵循了预期步骤？
  • 风格目标：输出是否遵循约定规范？
  • 效率目标：是否没有反复折腾地达成目标？

2. 创建 Skill

一个 Codex Skill 是一个包含 SKILL.md 文件的目录。使用 $skill-creator 快速脚手架。

3. 手动触发 Skill 以暴露隐藏假设

检查 Skill 是否在你期望的时候触发。暴露触发条件、环境和执行假设。

4. 使用小型、有针对性的提示词集尽早捕获回归

从 10-20 个提示词的小型 CSV 开始，混合以下类型：

  • 显式调用（"使用 xxx skill"）
  • 隐式调用（通过语境暗示）
  • 上下文调用
  • 负面控制组（不应触发的情况）

5. 使用轻量级确定性评分器快速入门

使用 codex exec --json，让 Eval 框架可以对实际发生的事情进行评分。针对 JSONL trace 中的 command_execution 事件编写确定性检查。

6. 使用 Codex 进行定性检查和基于 Rubric 的评分

使用 --output-schema 将最终响应约束为 JSON Schema，以实现一致的 Rubric 评分。

7. 随 Skill 成熟扩展你的 Evals

随着 Skill 发展，可以逐步添加以下检查项：

  • 命令数量和反复折腾次数
  • Token 预算
  • 构建检查
  • 运行时冒烟测试
  • 仓库清洁度
  • 沙箱和权限回归

8. Key takeaways

  • 衡量真正重要的事情
  • 从可检查的完成定义出发
  • 将 Evals 基于行为
  • 在规则不足之处使用 Codex
  • 让真实的失败驱动覆盖范围
