Why we built the Responses API

2025-09-22约 3 分钟Responses API架构
Summary
Responses API 将 Chat Completions 的单轮对话模式升级为结构化的 Agentic 循环：保留推理状态、支持托管工具、实现安全的跨轮续接。相比 Chat Completions，TAUBench 提升 +5%，缓存利用率提升 40-80%。
原文链接
https://developers.openai.com/blog/why-we-built-the-responses-api

Chat Completions 给了你一个简单的轮次对话界面。而 Responses 给了你一个用于推理和行动的结构化循环。Responses 在轮次之间保留模型的推理状态（TAUBench +5%，缓存利用率提升 40-80%）。

/v1/responses is an agentic loop

Chat Completions 是单轮对话的最佳工具。Responses API 则是为 Agentic 工作流设计的：

  • 保留模型在轮次之间的推理状态
  • 让模型在单次 API 调用中执行多个步骤
  • 以结构化方式处理工具调用和结果

Moving up the stack with hosted tools

File Search、Code Interpreter、网页搜索、图像生成、MCP——工具执行发生在服务端。这意味着：

  • 你不需要管理工具执行基础设施
  • 工具调用和结果在 API 内部处理
  • 支持更复杂的多步骤工作流

Preserving reasoning safely

Responses API 通过以下方式安全地保留推理：

  • 在内部加密存储推理状态，对客户端不可见
  • 通过 previous_response_id 实现安全的续接

Why /v1/responses is the best way to build

  • Agentic 工具使用：内置工具执行，无需手动协调
  • 默认有状态：跨轮次保留推理上下文
  • 从底层支持多模态：统一处理文本、图像、音频
  • 更低成本、更好性能：缓存利用率提升 40-80%
  • 更好的设计：语义化流式事件、内部标签多态、output_text 辅助方法

Chat Completions 不会消失，但 Responses API 是推理和 Agentic 工作流的前进方向。
