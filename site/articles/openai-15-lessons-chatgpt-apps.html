15 lessons learned building ChatGPT Apps

2026约 6 分钟ChatGPT AppsSDK实践经验
Summary
Alpic 团队三个月内开发了二十余款 ChatGPT 应用，总结出 15 条核心经验：构建 AI-first 应用与传统 Web 开发根本不同——需要管理「用户+UI+模型」三体问题、处理上下文不对称、重新设计 UI 交互范式，并将经验沉淀为开源 Skybridge 框架和 Codex Skill。
原文链接
https://developers.openai.com/blog/15-lessons-learned-building-chatgpt-apps

在 Alpic，我们相信下一代产品和服务将围绕 AI-first 体验构建——用户与模型协作而非导航传统预设 UI 流程的界面。OpenAI 发布 Apps SDK 后，我们立即开始构建。三个月内，我们为旅游、零售和 SaaS 等 B2B 和 B2C 场景开发了二十余款 ChatGPT App。

我们很快发现，构建 ChatGPT App 与传统 Web 或移动应用根本不同——在 Web 上行之有效的模式（即时数据获取、UI 驱动状态、显式用户配置等）在 Agentic 环境中往往失效，甚至会主动损害体验。

The three body problem

传统 Web 应用只有两个参与者：用户和 UI。而在 ChatGPT 应用中，第三个参与者进入了系统：模型。最难的部分是管理这三者之间的信息流动——我们称这种现象为上下文不对称。

1. 不是所有上下文都应该共享

字段用途可见范围
structuredContentWidget 和模型的类型化数据Widget 和模型均可见
_meta响应元数据仅 Widget 可见，对模型隐藏

2. 懒加载在 AI 应用中效果不佳

在 ChatGPT 中，范式是反转的：工具调用意味着延迟，由于安全沙箱和模型推理，往往需要数秒。我们学会了激进的前置加载：在初始工具响应中发送尽可能多的数据。

3. 模型需要可见性

使用 window.openai.setWidgetState(state) 存储特定状态数据，添加到模型的上下文中。我们还引入了声明式的 data-llm 属性：

<div
  data-llm={
    selectedTab === "details"
      ? "User is viewing product details"
      : "User is viewing reviews"
  }
>

4. 不同交互需要不同 API

ChatGPT App 涉及 Widget、服务器和模型之间多条交互路径，需要针对每种场景选择正确的 API。

Reinventing UI for AI

5. UI 必须适配多种显示模式

显示模式外观使用场景
Inline默认显示模式快速交互
FullscreenWidget 占满整个屏幕复杂 Widget
Picture-in-Picture与 Inline 同尺寸但浮动置顶后续对话中仍需保持可见

6. 嵌入式环境中的 UI 一致性至关重要

OpenAI Apps SDK UI Kit（基于 Tailwind CSS）提供了清晰的基准线。

7. 语言优先的筛选

不要用侧边栏的筛选排序选项，而是为工具参数提供值列表（LOV），让模型通过自然语言进行筛选。

8. 文件可以解锁更丰富的交互

体验可以从用户已有的文件开始，文件能开启全新的交互形式。

Going to production

9. CSP 是新的 CORS

OpenAI 在双层嵌套 iframe 中渲染 App。CSP 配置至关重要：

字段用途示例
connectDomainsAPI 和 XHR 请求https://api.weather.com
resourceDomains图片、字体、脚本https://cdn.jsdelivr.net
frameDomains嵌入 iframehttps://www.youtube.com
redirectDomains外部链接https://app.alpic.ai

10. 小的 Widget 标志有巨大影响

  • widgetDomain 是提交审核必填项
  • 工具注解必须遵循发布指南
  • widgetAccessible 控制 Widget 是否可以自行调用工具

Optimizing for fast iteration

11. 快速迭代需要热重载

我们构建了一个 Vite 插件，支持在 ChatGPT 内直接实时重载 Widget。

12. 并非每个测试都属于 ChatGPT

我们构建了一个轻量级本地模拟器来 mock ChatGPT 宿主环境。

13. 移动端测试需要明确支持

我们的 Vite 插件支持在隧道端口上进行域名转发，用于移动端测试。

14. 熟悉的抽象（如 React hooks）加速前端开发

我们引入了 useCallTool、useWidgetState、useLocale 等 hooks，以及基于 Zustand 构建的 createStore。

Turning lessons into a Codex Skill

15. 将经验转化为可复用工具

  1. Skybridge 框架：一个开源 React 框架，将 ChatGPT App 开发最佳实践封装为可复用抽象
  2. chatgpt-apps-builder Codex Skill：支持完整应用生命周期

安装方式：npx skills add alpic-ai/skybridge
