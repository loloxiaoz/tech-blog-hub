三条硬规则（最值得记忆的设计原则）

1. 先验证付费，再扩大投入——点赞、收藏、口头认可都不是验证；愿意付钱才是验证 2. 先人工跑通，再自动化——没有稳定流程时，不要急着上 Dify、n8n、MCP 或多 Agent 3. 先合规收款，再优化税负——不要用个人账户长期收业务款，不要为了节税制造虚假成本

这三条是新手最容易踩坑的三个领域。每一条都直接对应一个 Skill 的核心约束。

各 Skill 详解

opc-100day-profit（百日盈利总编排）

100 天三阶段：

阶段时间目标必须产出决策标准
1. 定位与付费验证Day 1-30找到有人愿意付费的具体问题赛道评分 / 10 人访谈 / MVP / 付费测试≥3 人愿意付费或已付费
2. 获客与交付闭环Day 31-70跑通获客/转化/交付/收款1-2 个主渠道 / 三档报价 / 5 个案例月收入 ≥¥3000，有可复用交付流程
3. 盈利与复购优化Day 71-100降低成本提高复购形成节奏成本表 / SOP / 复购方案 / 百日复盘收入覆盖运营成本和基本生活费

启动前 5 个假设（每个都要量化验证）：

  1. 你有可迁移能力（写下 3 个行业经验/技能/资源）
  2. 目标用户有痛点（访谈 10 人，至少 5 人承认）
  3. 目标用户愿意付费（提出 ¥99-199 付费测试）
  4. AI 能显著提效（3 天内做出可演示版本）
  5. 现金流能支撑试错（至少 3 个月生活费或兼职收入）

Go/No-Go 量化决策：

时间点Go延长验证No-Go
Day 7（赛道）总分 ≥80 且无单项硬伤60-79，需补访谈<60 或无明确付费人群
Day 14（访谈）≥5 人承认痛点且有替代方案痛点存在但付费人不清多数人只是"觉得有趣"
Day 30（付费）≥3 人愿意付费或已付费1-2 人愿意付费，延长 2 周0 人愿意付费
Day 60（增长）月收入 ≥¥3000，渠道可复用有收入但依赖熟人无自然咨询 / 无复购
Day 100（盈利）收入覆盖成本，SOP 可复制仍需兼职现金流支持方向、获客、交付都不可复制

执行原则：

  • 每天只抓一个 P0 任务
  • 不用"准备好了再发"，用真实反馈修正方向
  • 不把工具购买当作进展，只有客户反馈和收入才算进展
  • 前 30 天宁可慢，不要错；中间 40 天必须快；最后 30 天必须稳

详见 concepts/100-day-profit-blueprint。

opc-strategy（赛道 / MVP / 商业模式）

统一评分标准（赛道三维度，权重总和 100%）：

维度权重新手判断
需求刚性35%用户是否已经为这个问题花钱、花时间或承担损失
技术成熟度30%现有 AI 是否能稳定完成 70%+ 交付
变现清晰度35%30 天内是否能找到决策人并收第一笔钱
总分决策
≥80值得进入 7 天验证
60-79只做低成本验证，不投入开发
<60暂停或换赛道

硬性否决：变现清晰度 <60 分，或找不到明确付费人群——即使总分高也不要继续。

7 天付费验证流程：

天数动作产出
Day 1-2完成 10 人名单 + 5 个访谈痛点排序 + 原话摘录
Day 3-4做 MVP 或服务样例演示链接 / 样例报告 / 交付流程
Day 5-6向目标用户提出付费测试¥99-199 试用 / 定金 / 明确付款承诺
Day 7复盘付费意愿Go / 延长验证 / No-Go

商业模式选择：

形态适合新手吗适用场景
标准化服务最推荐AI 文案 / 数据分析 / 客服搭建 / 自动化咨询
模板/工具包推荐已有重复交付经验
订阅服务验证后推荐客户每月持续需求
纯 SaaS不建议一开始做开发/获客/客服/留存复杂度高
高度定制项目谨慎客单高但易困在服务陷阱

必须避免：

  • 为了"看起来像产品"先做官网、Logo、复杂系统
  • 用问卷代替真实访谈
  • 把免费试用人数当作付费验证
  • 进入自己没有行业理解、没有客户触达渠道的赛道

opc-compliance（注册 / 财税 / 版权 / AI 合规）

新手合规顺序：

  1. 先确认是否真的要收钱——没有付费验证前不要为"创业仪式感"注册公司
  2. 有第一笔 B 端订单时，优先合同和收款路径——金额、交付物、验收、退款、版权归属要写清
  3. 月收入稳定后建立主体——个人客户低价试水可延后；企业客户、开票、平台入驻通常需要主体
  4. 所有业务凭证归档——合同/发票/银行回单/交付记录/聊天确认都要保存
  5. 涉及用户数据或公开 AI 产品时做专项合规检查

主体选择：

情况建议理由
还在验证、收入不稳定暂不注册或个体户短期试水降低固定成本
企业客户、需要合同/发票/对公有限责任公司信任、风控、知识产权友好
个人接单、低风险、月收入低个体户可过渡成本低，但无限责任
有融资、合伙、股权计划先咨询律师/财税顾问架构错误后期成本很高
一人公司最大的风险不是"没注册"，而是注册后仍然公私不分。

财务防火墙（必须做到）：

  • 业务收入和支出尽量走对公账户或可解释账户
  • 不用公司账户支付个人消费
  • 所有收入有合同、订单或聊天确认
  • 所有支出有发票、账单、付款记录和业务用途说明
  • 每月固定一天整理凭证，至少保存 5 年

AI 业务红线：

场景风险新手动作
用客户数据训练或分析隐私和数据安全合同写明用途、范围、保存期限
AI 生成图片/文案/代码交付版权和侵权保留生成记录和人工审核记录
对外提供 AI 产品ICP / 算法 / 模型 / 内容安全上线前做专项合规检查
医疗、法律、金融建议专业责任风险明确辅助性质，必要时专家复核
抓取平台数据平台协议和数据来源不绕过权限，不采集敏感信息

何时必须找专业人士：

  • 单笔合同金额 >¥5000 且交付影响客户收入/合规/数据安全
  • 涉及退款、违约、侵权、投诉、客户数据泄露
  • 注册公司、申请核定征收、做股权设计、申请高新或软著
  • 产品面向公众开放，涉及算法推荐、深度合成、用户画像或个人信息处理

阶段目标动作指标
内容/触达让目标用户知道你解决什么每周 5-7 条痛点内容 + 主动触达 20 人有回复/收藏/私信

用户类型优先渠道内容形式

首单成交脚本（5 步）：

  1. 确认问题（你现在最想解决获客 / 转化 / 效率 / 成本？）
  2. 量化损失（这个问题每周大概浪费多少时间或多少钱？）
  3. 展示相似案例
  4. 给三档方案（低价试点 / 标准交付 / 深度定制）
  5. 设置下一步（今天确认诊断表 / 明天给方案 / 付款后进项目群）

交付后必须做：

  • 交付时附使用说明，不只丢文件
  • 3-7 天后问是否达成目标
  • 14 天后发额外洞察或优化建议
  • 30 天后给续费、升级或转介绍方案

opc-techstack（工具 / Agent / 自动化 / 成本）

新手工具栈 5 原则：

  1. 收入未验证前，月工具预算不超过 ¥500
  2. 每个工具必须服务一个业务动作（获客/转化/交付/复盘/收款）
  3. 先手工跑通流程，再自动化（流程不稳定时，自动化只会放大混乱）
  4. 每月做一次订阅审计，使用 <5 次或 ROI<3 的工具取消
  5. Agent 先做内部助手，再做对外自动回复（降低幻觉和承诺风险）

三层工具栈预算配置：

阶段月收入月预算推荐配置不建议
验证期<¥5000¥0-500免费模型 / 飞书/Notion / Canva / 表格自托管 / 复杂 Agent / 多平台付费
稳定期¥5000-20000¥500-2000Claude/ChatGPT / 一个设计工具 / 一个自动化工具同时买多套同类工具
增长期>¥20000¥2000+多模型 API / Dify/n8n / 监控 / 知识库没有成本上限的 API 调用

Agent 选型：

创始人情况先用原因
不会写代码，客户在微信/飞书Coze上手快，适合客服和内容助手
会一点技术，需要可控知识库Dify可视化工作流和 RAG 友好
需要连接多个 SaaSn8n自动化编排强
有工程能力，要深度定制LangGraph / LangChain灵活但维护成本高
MCP / A2A 属于进阶能力。新手只有在已经有稳定流程、明确工具调用需求、愿意维护配置时再引入。

成本控制底线：

指标健康值
AI 工具和 API 成本占收入15-25%
单个客户 AI 成本 / 客户收入<30%
缓存命中率>60%
低价模型调用占比≥70%
僵尸订阅每月清理

自动化优先级——优先自动化满足这些条件的任务：

  • 每周重复至少 3 次
  • 输入输出稳定
  • 出错代价低
  • 能直接提升获客/转化/交付/复盘效率
  • 手工流程已跑通至少 2 周

不要优先自动化：合同承诺、退款、投诉、复杂报价、高客单售前。
