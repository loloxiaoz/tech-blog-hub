Author: Wesley Pasfield | Date: May 12, 2026 | Source: OpenAI Cookbook

This notebook builds an improvement flywheel for an agent: traces → feedback → evals → optimization → Codex handoff.

The Improvement Flywheel

  1. Create an OpenAI Agents SDK-backed financial analyst
  2. Run it on synthetic company data and capture traces
  3. Add human and model feedback from those runs
  4. Turn that feedback into Promptfoo evals
  5. Use HALO to rank next harness changes and write Codex-ready handoff

Harness = full contract around the model (instructions, tools, routing, output requirements, validation checks).

What You Build

  • Agents SDK-backed financial analyst reviewing fictional company diligence materials
  • Human and LLM-generated feedback over traced runs
  • Automatically generated Promptfoo eval suite
  • HALO optimization pass over traces, feedback, and eval results
  • Developer-facing handoff to Codex for harness implementation

Step 1: Create Synthetic Company Data

FictionalCorp XYZ — revenue intelligence SaaS company with $43.0M ARR, 71% growth.

Synthetic Files:

  • overview.md — Management summary
  • product_strategy.md — Roadmap + unvalidated NRR estimate
  • go_to_market.md — Sales motion context
  • board_deck.md — Polished management narrative
  • financials/revenue_recognition_notes.md — Accounting context
  • legal/contracts_summary.md — Contract-level risk
  • legal/open_issues.md — Open legal matters
  • security/security_overview.md — SOC 2 status
  • sales/security_faq.md — Sales-facing security language
  • notes/qa_log.md — Diligence Q&A

Key Conflicts (designed for improvement):

  • Board ARR ($43.0M) ≠ Finance-controlled ARR ($36.9M)
  • NRR estimate (122%) is management estimate, not finance-validated
  • SOC 2 "complete" in sales FAQ ≠ SOC 2 Type II report status
  • Top-2 customer concentration = 34% (not 15% as management states)

Step 2: Define the Agents SDK-backed Analyst

Agent configuration (AgentConfig) with:

  • System prompt with evidence rules
  • Tool policy (allowed data root, required artifacts)
  • Model settings (gpt-5.5, reasoning_effort: medium)
  • Runtime validation tools

Required Artifacts:

  • outputs/summary_answer.md
  • outputs/investment_memo.md
  • outputs/risk_register.json
  • outputs/open_questions.md
  • outputs/citations.json
  • outputs/evidence_table.csv

Validation Tools:

  • data/tools/check_evidence_coverage.py — Audit drafted claims against cited dataroom files
  • data/tools/validate_output_contract.py — Validate required artifacts and JSON shape

Step 3: Generate Traced Runs

5 traced runs with questions covering:

  1. Runway and burn → financing risk
  2. Revenue quality and which ARR figure to rely on
  3. Real customer concentration risk after parent-account rollups
  4. Enterprise security review readiness
  5. Unsupported metrics to refuse inferring

Each run writes:

  • Full trace via HALO JSONL exporter
  • All 6 required artifacts
  • Agent's final answer

Step 4-8: Remaining Loop Steps

Step 4: Add Human and LLM Feedback — Structured feedback on trace outputs

Step 5: Generate Promptfoo Evals — Convert feedback into reusable eval suites

Step 6: Run Promptfoo Validation Gate — Test current agent against eval suite

Step 7: Run HALO Optimization — Analyze traces + feedback + eval results

Step 8: Generate Codex Handoff — Create codex_handoff.md with recommendations

Key Design Patterns

  • Trace export: Local HALO JSONL exporter from Agents SDK spans
  • Eval generation: Feedback → Promptfoo YAML configs
  • Optimization: HALO ranks harness changes by expected improvement
  • Handoff: Structured markdown for Codex to implement changes

Conclusion

The loop preserves what you learn from each run. Traces show what happened, feedback explains what mattered, evals make expectations reusable, and Codex acts on the resulting change set.
