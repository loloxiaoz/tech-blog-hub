Computer Use Agents in Daytona Sandboxes

2026-04-19约 10 分钟AgentComputer UseSandbox
Summary
本教程展示如何使用 OpenAI Agents SDK 的 Computer Use 工具配合 Daytona 沙箱，实现浏览器 UI 自动化操作。通过 DaytonaAsyncComputer 适配器将沙箱桌面接入 Agents SDK，智能体可以像人类一样截图、点击、输入，完成网页表单填写等任务，整个过程可录制为视频。
原文链接
https://developers.openai.com/blog/computer-use-agents-in-daytona-sandboxes

概述

很多有价值的工作仍然存在于没有公开API的浏览器UI中：第三方仪表盘、管理面板、表单密集型工作流。OpenAI Agents SDK 的 Computer Use工具允许智能体查看并控制桌面系统。在本教程中，我们使用 Daytona 沙箱作为桌面的运行载体。

Computer Use工具只需要几个基础原语即可驱动桌面：screenshot（截图）、click（点击）、type（输入）、scroll（滚动）、press keys（按键）。Daytona沙箱将包含浏览器的Linux桌面封装在Python SDK中，恰好暴露了这些原语。一个实现了Agents SDK AsyncComputer接口的轻量适配器可以将沙箱接入该工具。

智能体循环在本notebook中运行，而沙箱负责实际的点击和输入操作。作为演示，本教程中的智能体将填写一个网页表单。该表单在沙箱内的localhost:8080上提供服务，整个会话会被录制为.mp4视频。

相同的模式适用于任何可以描述为"打开应用、导航到某个位置、与屏幕交互"的任务：端到端UI流程测试、驱动传统桌面软件，或任何仅以人工交互界面存在的流程。

前置要求

  • Python 3.10+
  • Daytona 账号和API密钥，导出为环境变量 DAYTONA_API_KEY
  • OpenAI API密钥，导出为环境变量 OPENAI_API_KEY
  • OpenAI Agents SDK和Daytona Python SDK

请将两个API密钥保存在你的shell环境中。本notebook通过os.environ[...]读取它们，不会将密钥写入沙箱。

安装依赖

克隆cookbook仓库并进入本示例目录：

git clone https://github.com/openai/openai-cookbook.git
cd openai-cookbook/examples/agents_sdk/computer_use_with_daytona

从该目录打开 computer_use_with_daytona.ipynb，并安装以下依赖：

导入和环境配置

我们从三个地方导入依赖：

  1. Agents SDK（Agent、Runner、ComputerTool，以及我们要实现的 AsyncComputer/Button/Environment 类型）
  2. Daytona SDK（AsyncDaytona 和 CreateSandboxFromSnapshotParams）
  3. 标准库的异步/路径辅助工具
from __future__ import annotations

import asyncio
import logging
import os
from pathlib import Path
from typing import Any

from daytona import AsyncDaytona, CreateSandboxFromSnapshotParams
from agents import Agent, AsyncComputer, Button, ComputerTool, Environment, Runner, trace
from IPython.display import Video

assert os.environ.get("DAYTONA_API_KEY"), "DAYTONA_API_KEY is not set."
assert os.environ.get("OPENAI_API_KEY"), "OPENAI_API_KEY is not set."

logger = logging.getLogger("computer_use_with_daytona")

Computer Use 适配器

Agents SDK的 Computer Use工具 可以对接任何实现 AsyncComputer 接口的对象：该接口要求有一个返回base64编码PNG的 screenshot 方法，以及 click、double_click、scroll、type、keypress、move、drag、wait 方法。

Daytona的桌面沙箱在 sandbox.computer_use.* 下暴露了匹配的API。下面的类就是两者之间的适配器：

_DEFAULT_WIDTH, _DEFAULT_HEIGHT = 1024, 768

_CUA_KEY_TO_DAYTONA: dict[str, str] = {
    "arrowdown": "down",
    "arrowleft": "left",
    "arrowright": "right",
    "arrowup": "up",
    "option": "alt",
    "super": "cmd",
    "win": "cmd",
}

def _normalize_key(key: str) -> str:
    if len(key) > 1:
        key = _CUA_KEY_TO_DAYTONA.get(key.lower(), key.lower())
    return key

class DaytonaAsyncComputer(AsyncComputer):
    """基于Daytona沙箱桌面的AsyncComputer实现。"""

    def __init__(self, sandbox, *, width=_DEFAULT_WIDTH, height=_DEFAULT_HEIGHT):
        self._sandbox = sandbox
        self._width = width
        self._height = height

    async def __aenter__(self):
        await self._sandbox.computer_use.start()
        await asyncio.sleep(2)
        return self

    async def __aexit__(self, exc_type, exc_val, exc_tb):
        try:
            await self._sandbox.computer_use.stop()
        except asyncio.CancelledError:
            raise
        except Exception:
            logger.warning("Failed to stop computer-use processes", exc_info=True)

    @property
    def environment(self):
        return "ubuntu"

    @property
    def dimensions(self):
        return (self._width, self._height)

    async def screenshot(self):
        response = await self._sandbox.computer_use.screenshot.take_full_screen()
        return response.screenshot or ""

    async def click(self, x, y, button):
        if button not in ("left", "right"):
            return
        await self._sandbox.computer_use.mouse.click(x, y, button)

    async def double_click(self, x, y):
        await self._sandbox.computer_use.mouse.click(x, y, "left", True)

    async def scroll(self, x, y, scroll_x, scroll_y):
        if scroll_y != 0:
            direction = "down" if scroll_y > 0 else "up"
            amount = max(1, abs(scroll_y) // 100)
            await self._sandbox.computer_use.mouse.scroll(x, y, direction, amount)

    async def type(self, text):
        await self._sandbox.computer_use.keyboard.type(text)

    async def wait(self):
        await asyncio.sleep(1)

    async def move(self, x, y):
        await self._sandbox.computer_use.mouse.move(x, y)

    async def keypress(self, keys):
        if not keys:
            return
        if len(keys) == 1:
            await self._sandbox.computer_use.keyboard.press(_normalize_key(keys[0]))
        else:
            *modifiers, key = keys
            await self._sandbox.computer_use.keyboard.press(
                _normalize_key(key), [_normalize_key(m) for m in modifiers]
            )

    async def drag(self, path):
        if len(path) < 2:
            return
        for i in range(len(path) - 1):
            sx, sy = path[i]
            ex, ey = path[i + 1]
            await self._sandbox.computer_use.mouse.drag(sx, sy, ex, ey)

表单、数据和提示词

我们要填写的表单位于本文件夹的 form.html 中。它是一个单页HTML注册表单，包含5个字段集：个人信息、职业详情、会议偏好、旅行/住宿、附加信息。字段涵盖文本输入、邮箱、电话、日期、<select> 下拉框、单选组、带最大选择限制的多选复选框组和一个文本域。

我们通过三部分告诉智能体要做什么：

  1. APPLICANT_DATA：智能体必须输入的信息，从 fake_applicant_data.txt 加载。
  2. INSTRUCTIONS：系统提示，填写表单的指导说明。
  3. TASK：启动运行的简短用户输入。
_DESKTOP_SNAPSHOT = "daytonaio/sandbox:0.6.0"
_FORM_DIR = "/home/daytona/form"
_SERVER_PORT = 8080

APPLICANT_DATA = Path("fake_applicant_data.txt").read_text()

INSTRUCTIONS = """\
You control a remote Linux desktop via mouse, keyboard, and screenshots.
A conference registration form is being served at http://localhost:8080.

When asked to fill the form:
1. Open a browser (look for one in the taskbar or application menu).
2. Navigate to http://localhost:8080.
3. Fill every field using the applicant data the user provides.
4. Click "Complete Registration".
5. When you see the "Registration Complete!" confirmation, take a screenshot
   and say DONE.
"""

TASK = f"Fill the conference registration form with this applicant data:\n\n{APPLICANT_DATA}"

创建沙箱

AsyncDaytona() 从 DAYTONA_API_KEY 读取API密钥。daytona.create(...) 从桌面快照启动沙箱；调用在沙箱准备好进行文件系统和进程操作时返回。沙箱句柄将传递给下游所有组件。

daytona = AsyncDaytona()
sandbox = await daytona.create(
    CreateSandboxFromSnapshotParams(snapshot=_DESKTOP_SNAPSHOT),
)
print(f"Sandbox ready: {sandbox.id}")

在沙箱内启动表单服务

我们将 form.html 上传到沙箱，并使用 python3 -m http.server 在8080端口提供服务。注意两个细节：

  1. Daytona Python SDK上传的是字节流而非路径，因此我们在主机上读取 form.html，将字节流推送到 /home/daytona/form/index.html。
  2. 后台运行的服务器需要将stdout/stderr重定向到日志文件，以便 exec 立即返回。
form_html = Path("form.html").read_bytes()
await sandbox.fs.create_folder(_FORM_DIR, "0755")
await sandbox.fs.upload_file(form_html, f"{_FORM_DIR}/index.html")

await sandbox.process.exec(
    f"sh -c 'cd {_FORM_DIR} && python3 -m http.server {_SERVER_PORT} "
    f"> /tmp/httpd.log 2>&1 &'"
)

运行智能体

这是核心步骤。我们执行以下操作：

  1. 进入 DaytonaAsyncComputer 上下文管理器，启动沙箱的computer-use进程（Xvfb、窗口管理器、VNC）。
  2. 启动会话录制。
  3. 构建 Agent，仅配置一个工具 ComputerTool(computer=computer)，在 trace(...) 块中使用 Runner.run(...) 运行它。
  4. 在 finally 块中，停止录制并将其下载到本地。
async with DaytonaAsyncComputer(sandbox) as computer:
    recording = await sandbox.computer_use.recording.start("form-fill")
    print(f"Recording started: {recording.id}")

    try:
        with trace("Daytona form-fill demo"):
            agent = Agent(
                name="Form filler",
                instructions=INSTRUCTIONS,
                tools=[ComputerTool(computer=computer)],
                model="gpt-5.4",
            )
            result = await Runner.run(agent, TASK, max_turns=50)
            print(f"\n--- Final output ---\n{result.final_output}")
    finally:
        stopped = await sandbox.computer_use.recording.stop(recording.id)
        local_recording_path = Path(stopped.file_name).name
        await sandbox.computer_use.recording.download(recording.id, local_recording_path)
        print(f"Recording downloaded to: {local_recording_path}")

清理沙箱

使用完毕后删除沙箱。Daytona也会按自己的计划自动销毁沙箱，但显式删除可以保持账号整洁，并确保不会产生闲置时间的费用。

await daytona.delete(sandbox)
print(f"Sandbox {sandbox.id} deleted.")

后续扩展方向

  • 其他类型的表单：该模式适用于智能体可以通过浏览器访问的任何网页表单。可以替换 form.html 为其他页面，或者去掉HTTP服务器，将智能体指向外部URL。
  • 其他后端：AsyncComputer 是可移植的接口。如果你将适配器和沙箱替换为其他支持CUA的桌面，本notebook的其余部分保持不变。
  • 评估：验证可以在沙箱内运行：将提交的负载与 APPLICANT_DATA 对比，你就可以得到一个确定性的表单填写评估。
