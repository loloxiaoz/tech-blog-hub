## 关键术语创新（最重要）

| 术语 | 定义 | 与本 wiki 已有术语关系 |
|---|---|---|
| **OPC（One Person Company / 一人公司）**| 政策侧/媒体侧的标准用语 | 本 wiki 已有 concepts/opc |
| **OPE（One Person Enterprise / 一人企业）**| Easy v2.0 改名：业务体（不必注册公司），以个体或个人品牌为主导 | OPC 的体系化升级表达 |
| **OPB（One Person Business / 一人业务）**| Easy v2.1 引入：一人企业下的具体业务单元 | **新维度** |

**v2.0 → v2.1 的核心升级：一人企业 ≠ 一人业务**
- 之前思路：项目 = 企业，一旦业务失败就丢掉整个企业
- 升级后：企业是**资产池**，业务是池中条目；**业务可死，企业不死**；多业务协同复用资产
- 工程含义：用统一品牌、统一用户系统、统一后端 API 串起多个业务（参考 Easy 的"方糖品牌+二级域名"实践）

## 三个里程碑（成长路径）

> 螺丝钉 → **存活个体** → **一般个体** → **超级个体**

| 节点 | 定义 | 状态 |
|---|---|---|
| 存活个体 | 达到**最低可行利润 MVPr**——副业/独立项目利润持平生活开销 | 伪-财务自由（可能需要降低生活成本配合）|
| 一般个体 | 副业/独立项目收入 ≥ 主业收入 | **工作自由**——不上班也有持平的被动收入 |
| 超级个体 | 真正财务自由，无需降低生活标准 | Easy 自评尚未达到 |

**MVPr** 是一个反复出现的关键概念：达到这个利润阈值就有了"长期主义"的奢侈。

## 方法论结构（4 模块 28 节）

### 模块一·定义（2 节）
- 新版方法论概述：v2.0 改"公司"为"企业"避免与公司法的"一人有限责任公司"混淆
- 一人企业的定义：**业务体（不必注册公司）/ 个体或个人品牌主导 / 一人企业 ≠ 个体户 ≠ 创业公司**
- 三大特点：员工少（成本低/灵活/知识技能受限）/ 资源有限（时间稀缺，半年试错；零成本推广，地狱开局；被迫精简和专注）/ 以小博大（多重竞争 / 成功一次即可的游戏 / **认知层的竞争**）

### 模块二·规划（8 节）

#### 4 条底层逻辑
| # | 主题 | 核心 |
|---|---|---|
| 1 | **以小博大** | **边界变动**理论：技术/信息传播/政策三类边界变动带来颠覆机会；《创新者的窘境》价值网络冲突让大公司"看得见做不了"|
| 2 | **规模化** | **新杠杆**（纳瓦尔）：代码 / 媒体 / **元杠杆-AI** 三类零边际成本杠杆 + 新基础设施（外包/众包/以租代建）|
| 3 | **资产和被动收入** | 出售时间片不可能规模化（一人上限月入 4 万）；解法是持有"能把钱放进口袋"的资产；获取路径：富则投资收购，穷则自行创造（数字商品 / NoCode / 开源+AI / 自学编程）；**资产五维量化评估**：投入 / 产出 / 持续性 / 风险 / 门槛 |
| 4 | **滚雪球与链式传播** | 复利：用资产收益滚动新资产；**搞定二次传播 = 省下市场费用** |

#### 赛道选择
- 远离大众刚需，选择**小众刚需**——大众弱需求但小众强需求
- 关键不是市场大小，而是能否**低成本聚集和触达**目标人群
- 适合一人企业的赛道：SaaS / 内容创作 / 教育 / 社区 / AI 等细分领域

#### 2 条竞争策略
| 策略 | 核心 |
|---|---|
| **不竞争策略** | 打不过就加入（成为生态的一部分：WordPress 插件年收入 5700 万美元）/ 选择**非标准化**或**弱替代性**品类 / 通过个性化或改变商业逻辑来改变品类属性 / 跨界混搭**创造新品类** |
| **结构化优势** | 针对竞品的**固有弱点**（来自商业模式或竞争优势的路径依赖）构建优势：副产品优势 / 第三方优势（聚合套路）/ 低成本优势（长期主义、剩者为王，Pinboard 收购 Delicious 案例）/ 专属服务优势 |

#### 思考工具：一人企业画布 + 一人企业月报
- **OPB 画布**（基于商业模式画布改造）：新增**竞争策略 / 资产评分 / 原有方案痛点 / 解决方案** 4 个独有模块
- **一人企业月报**：月初设定（收益目标 vs MRR vs MVPr vs 工作自由 MRR）+ 资产变动 + 三池状态（用户池 / 内容池+基础设施 / 产品池）；月末复盘（资产现状/处置 + 收益目标对照）
- 月报作用：**强制性以全面角度思考业务，定期进行，防止漏掉重要的部分**

### 模块三·构建一人业务（5 节）

| 主题 | 核心 |
|---|---|
| OPE ≠ OPB | 一人企业可包含多个一人业务；不为盈利的业务（蹭热点）也有用——为后续业务降低获客成本 |
| **副产品优势**（优势发现）| 主业副产品（工作流/方法论 → 书 / 课程 / 电子书 / 长文 / Custom GPT / AI workflow / 产品化；行业知识库 → 调研报告 / AI 聊天机器人；备选方案/半成品 → 公益化 or 转换形式；人脉 → 撮合 / 付费社群）+ 生活方式副产品（湾区日报案例 → AI 重构为 Reading as a Service）+ 兴趣爱好副产品 |
| **从副业开始**（风险评控）| 用主业保最低收入，副业拓天花板；双线收入提供财务+职业双重安全 |
| **管理利用不确定性** | 不确定性前置（MVP / 落地页 / 众筹）；**反脆弱性**（小步快跑/冗余/去中心化/保留选择权）；**不对称交易**（有限损失，高额收益）|
| 软件构建 | 价值主张 → 画布 → 用户画像 → 场景 → 功能分期 → MVP → 众筹 → 迭代 |

### 模块四·基础设施（6 节）

#### 为什么要自建基础设施
- **平台商业模式**会限制你（小号没流量、大号被削弱、扶持期后流量回收）
- **封号风险**——单点崩溃，多年积累化为乌有
- **产品形态/品牌约束**——平台不允许独立域名、不开放接口
- **最佳实践**：自有基础设施 + 多平台分发 + 用户回流到自有平台 = 混合生态

#### 理想基础设施 6 大要求
**足够便宜**（用户量小时几乎免费）/ **足够可控**（不被平台决定）/ **个人可用**（无需注册公司即可运营）/ **足够开放**（可二次开发）/ **繁荣生态**（插件复用降低成本）/ **适配方法论**（开箱即用支持 OPB 流程）

#### 三池四能力架构

| 三池 | 四能力 |
|---|---|
| 用户池（注册登录集中管理；微信登录是国内最低成本方案）| 触达能力（APP 推送 / 短信 / 邮件 / 群消息 / 微信模板，群消息和模板效果最好）|
| 内容池（官网作为品牌入口和风险防护）| 自动化能力（跨平台自动发布；AI 聊天驱动工作流）|
| 产品池（多个可插拔业务组件，共享用户/内容）| 支付能力（个人可通过 XorPay 等小微支付低成本接入；自建收银台支持无限业务扩展）|
| | **众包能力**（任务分包给用户，需要：清晰细分 / 量化验收 / 有吸引力奖励；裂变/推荐返现是垂直众包的典型形式）|

#### 落地建议
- 初期用 **WordPress** 做 MVP/众筹（一年服务器费用 50 元以内）
- Easy 团队在做 **方糖 OPB 插件**（WordPress 插件）+ BudCoder + FlowDeer，目标是开箱即用支持本方法论
- 即使自行开发，也可以从三个层次和 WordPress 混搭

## 关键金句

> "**一人企业的核心是维持一个能不断带来被动收入的资产池，需要定期对其中的资产进行维护和重配置。**"
>
> "**新杠杆并不能取代传统杠杆，它们只能在数字经济领域呼风唤雨。一旦涉及到实体，依然需要传统杠杆。所以没有必要将它们对立起来，我们可以全都要。**"
>
> "**最好的产品，不是最好的产品。**"——意指市场不一定选择技术最优而是综合最优
>
> "**一人企业的竞争，有一半都是在认知上。**"
>
> "**搞定了二次传播，也就省下了市场费用。**"
>
> "**这是成功一次即可财务自由的游戏。**"
>
> "**我们要做的东西在早期应该尽可能的少，但是思考的范围却应该尽可能的广。**"
