# 蔚来销售大模型的工程应用与优化实践

**来源**：QCon 全球软件开发大会（InfoQ 极客传媒）演讲 PPT
**演讲者**：邰康盛，蔚来汽车
**主题**：蔚来销售大模型从样本生产到训练到上线的完整工程实践，含 GRPO 强化学习 + B 端小样本 AB 实验框架

---

## 一、业务背景与核心痛点

### 蔚来销售场景

- **三品牌**：NIO / 乐道 ONVO / 萤火虫 Firefly
- **销售 6 阶段**：认知 → 兴趣 → 意向 → 洽谈 → 成交 → 交付（全流程跟进）

### 两大核心痛点

1. **新人与销冠效果差异大**：隐性知识难以传递，销冠经验无法规模化复制
2. **跟进工作量大，单兵销能受限**：用户旅程长，销售需要记忆大量上下文和策略

---

## 二、智能销售三阶段演进

| 阶段 | 特征 | 当前状态 |
|------|------|---------|
| **阶段一** | 单点提效工具，小模型主导，独立功能 | 已完成 |
| **阶段二（当前）** | 销售 Agent-1.0，知识智能体，AI 提供决策支持 | **进行中** |
| **阶段三（目标）** | 销售 Agent-2.0，自主执行，AI 独立完成跟进动作 | 规划中 |

### 当前 Agent（阶段二）能力输出

```
回忆认知（生成用户总结）
  ↓
大脑决策（生成跟进策略）
  ↓
跟进执行（推荐物料）
```

**3 设计原则**：使用者视角 / 可阅读性 / 层层递进（先概要后细节，不直接把raw数据扔给销售）

## 三、技术架构三层

### 训练侧

- 销售样本集生成（自动化流水线）
- 销售大模型垂类训练（SFT微调 + GRPO强化学习）
- AB 在线实验框架

### 推理侧

**数据层**：用户数据 / 销售知识库 / 上下文工程 / 数据脱敏

**Agent 层**：模型 Hub / RAG 知识引擎 / 提示词管理 / 工具库管理

**应用层**：调度引擎 / 风控模块 / 日志管理 / 埋点反馈

## 四、关键实践1：高效的样本生产线

### 4.1 销售知识萃取

```
销冠通话记录 + 最佳实践 + 带教反馈
  ↓
销售策略案例库
  ↓ 二级标签：销售阶段 + 用户卡点 → 销售策略
```

### 4.2 评估 Agent 一致性优化

挑战：评估一致性不足，人机标注差距大。

解决流程：
1. 制定黄金评测集（含关键要点和错误要点）
2. 机器初次评估
3. LLM 自动优化评估器提示词（用黄金评测集作为反馈）
4. 迭代至一致率 **>90%**

### 4.3 样本库治理：5维标签体系

| 维度 | 说明 |
|------|------|
| 用户销售阶段 | 认知/兴趣/意向/洽谈/成交/交付 |
| 用户购车卡点 | 价格/竞品/交付期/服务等 |
| 场景风险度 | 高/中/低（影响质量门槛） |
| 上下文丰度 | 信息量多少（影响生成难度） |
| 样本困难度 | 容易/困难（用于课程学习） |

多模型生成 + 多模型评估 + 统计分析，确保样本多样性。

## 五、关键实践2：目标导向的大模型训练

### 5.1 两阶段训练路径

```
SFT 微调（监督微调）
  目的：模仿销售专家表达，习得基本格式和风格
  数据：混合样本（垂类 + 通用，保持通用能力不退化）
  ↓
GRPO 强化学习
  目的：基于混合奖励函数持续优化
  机制：随机生成多个候选结果，奖励函数排序后强化
```

### 5.2 GRPO 混合奖励设计

三类奖励叠加：

| 奖励类型 | 内容 |
|---------|------|
| **格式规范奖励** | 输出格式符合规范（JSON结构/字段完整性等） |
| **过程奖励** | 策略建议质量 + 推理通顺性（过程合理） |
| **结果奖励** | 有标准答案：与标准答案对比；无标准答案：偏好评估 |

### 5.3 多阶段纠错 SFT

迭代提升错误 case 通过率的 5 种手段：
1. 错误样本回收，分析失败模式
2. 针对性构造纠错样本
3. 多轮混合训练（错误样本+正例样本）
4. 验证集跟踪，确保修复不引入退化
5. 迭代直到通过率满足标准

### 5.4 垂类模型 vs 商业模型对比

| 模型 | 用户总结准确率 | 策略建议有效性 |
|------|--------------|--------------|
| 商业大模型 Q | 73.0% | 68.0% |
| 商业大模型 K（T0基座） | 69.0% | 66.5% |
| 商业大模型 H | 71.0% | 72.5% |
| qwen3-14b-base | 46.5% | 44.0% |
| **qwen3-14b-sft_grpo** | **82.0%** | **82.0%** |
| qwen3.5-9b-base | 52.5% | 46.0% |
| **qwen3.5-9b-sft_grpo** | **83.5%** | **87.5%** |

**结论**：9B/14B 垂类模型经 SFT+GRPO 后，全面超越 T0 级商业大模型。小模型 + 垂域训练 > 大通用模型。

### 5.5 大模型后训练的适用边界

**适合垂类训练的场景**：
- 提示词工程 + RAG 优化后效果仍不满足需求
- 知识变化频次低（蔚来仅用于用户总结和策略建议场景）

**不适合垂类训练的场景**：知识更新频繁（应用 RAG）、通用推理任务

## 六、关键实践3：小样本场景下的 AB 评估框架

### B 端实验的核心挑战

C 端实验（百万用户）：大数定律保障随机分流可比。

B 端实验（销售：最多万级 FL）：
- **纯随机分流失效**：样本量太少，随机分组后实验组和对照组能力分布不均
- **业务价值被稀释**：样本过少导致统计检验无效

### 解决方案：基于销售能力模型的 AB 框架

**分流策略（实验前）**：
```
Step1：筛选地区（城市公司能力模型）
Step2：筛选 FL（销售能力模型 + 配对/分层抽样）
Step3：AA 检验（验证分组是否可比）
→ 生成实验组 FL 名单 + 对照组 FL 名单
```

**分析框架（实验后）**：
```
Step1：异常剔除
Step2：DID 分析（差分差分法，控制时间趋势）
Step2：T 检验（验证差异显著性）
```

**销售能力模型三维度**：
- 意愿与基础沟通能力（衡量是否用系统）
- 邀约到店能力（中间漏斗）
- 临门一脚的成交能力（最终转化）

**效果**：实验无系统性偏差，实验组和对照组在多个指标差异不显著（分组成功）。

## 核心论点总结

1. **垂类小模型 > 通用大模型**：9B+SFT+GRPO 的准确率全面超越 T0 级商业大模型，前提是高质量样本和正确训练策略
2. **B 端 AB 实验需要能力感知分流**：随机分流在小样本场景失效，需要用销售能力模型做配对/分层抽样
3. **样本质量 = 模型上限**：5维标签体系 + 评估一致性 >90% + 多模型多样性，是整个训练管线的基础
4. **GRPO 混合奖励**：格式+过程+结果三类奖励叠加，解决销售场景"无唯一标准答案"问题
