How Perplexity Brought Voice Search to Millions Using the Realtime API

2026-03-25约 4 分钟Realtime API语音Perplexity
Summary
Perplexity 使用 OpenAI Realtime API 将语音搜索扩展到数百万用户，核心挑战是上下文管理策略、跨平台音频标准化、真实噪声环境下的 VAD 调优，以及将工具集精简到十个以内。
原文链接
https://developers.openai.com/blog/how-perplexity-brought-voice-search-to-millions

1. Figure Out Your Context Management Strategy

不要使用大块内容，而是将所有内容分解成更小的 2,000 token 块并增量输入。当截断发生时，它会裁掉一点历史记录，而不是全部清空。

不是所有上下文都应以相同方式进入模型。正确使用 item.type: "message" 角色：

  • system：用于指令
  • user：用于终端用户输入
  • assistant：用于模型生成的输出

2. Standardize Audio Across Product Surfaces

Perplexity 用 Rust 构建了一个 SDK，抽象掉平台特定差异：重新采样至 48 kHz 单声道，通过 WebRTC APM 进行回声消除、自动增益控制（AGC）、降噪、高通滤波。

3. Tune for the Messy Environment

针对真实麦克风、扬声器音量和背景噪声校准 VAD（语音活动检测）。

声音锁定（Voice lock）：交互默认保持环境感知状态，但用户可以锁定语音以占用发言权。

4. Use Only Core Tools and Keep Them In Distribution

将工具集精简到十个以内。将工具输出格式化为清晰分隔字段的结构化 JSON：

// Good:
{
  "response_text": "I kicked-off the task to create a market research dashboard",
  "require_repeat_verbatim": true
}

这种格式让模型更容易准确地朗读响应，同时维护结构化的元数据。
