在 Amazon Bedrock 上开始使用 OpenAI 模型

2026-05-31约 12 分钟Amazon BedrockAPIResponses API
Summary
详细介绍如何在 Amazon Bedrock 上使用 OpenAI 兼容的 Responses API，涵盖配置 Bedrock 环境、验证端点、结构化输出、函数工具调用、直接 PDF 输入、有状态对话和提示词缓存等完整工作流。
原文链接
https://cookbook.openai.com/examples/partners/AWS/openai_models_with_amazon_bedrock

Amazon Bedrock 上的 OpenAI 模型提供了兼容 OpenAI 的 Responses API 接口，支持生产级工作流，包括文本生成、结构化输出、应用工具调用、直接文件输入、响应状态管理、提示词缓存和后台任务处理。本指南以虚构零售商 BrightCart 的售后支持助手工作流为例，演示完整的 API 使用流程。

你将学到

  1. 使用 Bedrock 专属环境变量配置 Bedrock 托管的 OpenAI 模型
  2. 验证 Responses 端点，检查响应 schema、用量元数据和标准化错误
  3. 使用原始 HTTPS 和 OpenAI SDK 发送文本请求
  4. 生成带 schema 约束的 JSON 和轻量 JSON 模式输出
  5. 调用应用管理的函数工具、并行工具和自定义文本工具
  6. 发送直接 PDF 输入、维护有状态和无状态对话、携带加密推理上下文
  7. 使用提示词缓存、后台模式、上下文压缩、运维检查和存储响应清理

前置要求

  • Amazon Bedrock 上 OpenAI 模型的 Bearer 令牌
  • Python 3.9 及以上版本
  • 可访问 Bedrock OpenAI 兼容端点的网络环境

支持的模型

AWS 区域支持的模型 ID
us-west-2openai.gpt-5.4
us-east-2openai.gpt-5.5、openai.gpt-5.4

1. 配置 Amazon Bedrock

运行 Notebook 前请先设置以下环境变量，默认配对为 us-west-2 + openai.gpt-5.4：

export AWS_BEARER_TOKEN_BEDROCK="YOUR_BEDROCK_BEARER_TOKEN"
export AWS_REGION="us-west-2"
export BEDROCK_MODEL="openai.gpt-5.4"
export BEDROCK_BASE_URL="https://bedrock-mantle.${AWS_REGION}.api.aws/openai/v1"

Bearer 令牌从 AWS_BEARER_TOKEN_BEDROCK 读取，如果缺失，配置单元格会弹出口令式提示，不会明文打印令牌。

安装依赖

%pip install -U "openai>=2.28.0" requests pandas ipython --quiet

创建客户端

from openai import OpenAI
client = OpenAI(api_key=AWS_BEARER_TOKEN_BEDROCK, base_url=BEDROCK_BASE_URL, max_retries=0)

2. 发起第一次 Responses 请求

本节从两个角度展示 Responses 请求接口：首先检查和运行原始 HTTPS 请求，让端点、请求头和 JSON 体可见；然后使用 OpenAI SDK 发起相同的应用工作流。

验证端点

第一次实际调用故意设计得非常简单：发送一个 store=false 的最小 Responses 请求，在运行更复杂的示例前先排查配置问题。

preflight_payload = {
    "model": MODEL_ID,
    "input": "Reply with exactly: ok",
    "max_output_tokens": 1024,
    "store": False,
}

使用 OpenAI SDK

sdk_text_payload = {
    "model": MODEL_ID,
    "input": "Write a three-sentence overview for a developer building a BrightCart support assistant with the Responses API.",
    "reasoning": {"effort": "low"},
    "max_output_tokens": 1024,
    "store": False,
}
text_response = create_response(**sdk_text_payload)

3. 生成结构化 JSON

结构化 JSON 将模型输出转换为应用代码可以解析、验证和路由的数据。本节对比严格的 schema 约束输出和轻量 JSON 模式：

  • Structured Outputs：当需要严格契约时使用，通过 text.format.type="json_schema" 和 strict=true 约束输出
  • JSON 模式：当只需要有效 JSON 但精确 schema 可以灵活时使用，通过 text.format.type="json_object" 指定

4. 添加应用管理的工具

函数调用让模型可以向应用请求数据或操作，但你的代码仍然负责执行工具并返回结果。本节定义本地 BrightCart 工具，演示：

  • 单个函数调用：模型选择参数，应用执行本地函数，返回 function_call_output
  • 并行工具调用：允许模型在一次轮次中请求多个独立查询，设置 parallel_tool_calls: True

5. 直接 PDF 输入

Responses API 可以接受直接 PDF 输入，以便模型可以从上传的文档中提取、汇总或回答有关问题。通过 input_file 类型发送内联 base64 数据或可访问 URL：

{
    "type": "input_file",
    "filename": "policy.txt",
    "file_data": f"data:text/plain;base64,{policy_b64}",
}

6. 维护对话状态

有状态对话让应用可以在多个轮次中携带上下文：

  • 有状态对话：使用 previous_response_id 链接每个响应到先前存储的响应 ID
  • 无状态对话：每次请求时完全控制历史记录，显式传入历史消息列表

7. 高级功能

  • 提示词缓存：通过 prompt_cache_key 和 prompt_cache_retention 减少重复前缀的 token 消耗
  • 后台模式：通过 background: True 异步处理长时间运行的任务
  • 上下文压缩：通过 context_compression 控制长对话的上下文管理策略
  • 存储响应清理：通过 responses.delete(response_id) 清理已存储的响应

结论

Amazon Bedrock 上的 OpenAI Responses API 提供了与原生 OpenAI API 一致的接口，让开发者可以在 AWS 基础设施上无缝使用 OpenAI 模型的完整功能集，同时享受 Bedrock 的企业级安全和合规保障。
