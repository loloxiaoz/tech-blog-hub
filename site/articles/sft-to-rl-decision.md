# 大厂实战中，如何判断SFT到什么程度开始做RL

## 核心定位

> **SFT 负责"教规矩"（学套路），RL 负责"优选"（比好坏）。**

SFT = Token-level 模仿，让模型达到数据集中"平均专家"水平。
RL = Sequence-level 评价，允许模型通过自主探索超越训练集。

## 何时停止 SFT，进入 RL

### 信号一：SFT 已充分
- 格式和能力已"对齐"：能稳定遵循指令（JSON 格式输出、识别语言、不答非所问）
- 性能遇到瓶颈：loss 降到低位不再波动，但效果提升不明显 → 已接近数据集天花板

### 信号二：RL 有充裕提升空间
- **pass@k 明显高于 pass@1**：模型多次生成时通过率显著高于单次 → 说明"会但不稳"，RL 可以引导模型第一次就生成高概率正确答案
- **SFT 出现幻觉与过拟合**：开始机械背诵训练集事实，未见过的问题上泛化性变差

> 研究表明：SFT 容易导致记忆，而 RL 有助于泛化。

## 为什么必须接 RL

| 场景 | 原因 |
|------|------|
| 需要通过"负反馈"纠偏 | SFT 教"应该说什么"，很难教"不该说什么"；RL 的惩罚项能有效抑制有害输出 |
| 任务没有唯一标准答案 | SFT 只能模仿某一样本；RL 通过 Ranking 告诉模型哪个回答更好 |
| 追求"卓越"而非"合格" | SFT 上限由数据集质量决定；RL 允许在探索中发现比训练集更好的表达 |

## SFT vs RL 分工表

| 维度 | SFT | RL |
|------|-----|----|
| 主要目标 | 指令遵循、格式掌握 | 偏好对齐、质量优化、安全性 |
| 学习方式 | Token-level 模仿 | Sequence-level 评价 |
| 数据需求 | 高质量"指令-回答"对 | 同一问题的"好/坏"排序对 |
| 效果表现 | 保证"下限"（不乱说话） | 提升"上限"（说得更好） |

## 行业主流实践路径

```
冷启动 SFT（几千~几万条高质量样本）
  → Reasoning RL（逻辑题：代码运行/数学验证作为奖励）
  → Preference RL（主观题：Reward Model 或人类排序）
  → 反思微调（SFT on RL 筛选的高质量样本）
  → 螺旋式迭代
```
